## 關鍵發現(重要性排序)

1. **舊 marker 無 lang 屬性 → 一律視為 zh-TW** 是整個相容策略的核心:0.9 之前只存在中文安裝,這條推定讓三個既有 repo 的 update 全部零參數正確沿用語言。marker 自 v0.9.0 起帶 `lang=xx` 屬性。
2. **hook 的語言與版本用「裝機蓋章」解**(cli 複製時字串替換 `const LANG` / `const FW_VERSION`):安裝副本保持零依賴單檔,不需 config 檔;statusline 的更新提示因此能比對本機版本與 npm latest。
3. **更新檢查的安全設計**:渲染永不碰網路——讀 tmpdir 快取,過期 24h 才 spawn detached 背景刷新(先 touch 時間戳防並發重複 spawn);快取放 tmpdir 而非 repo(否則 git status 髒掉,儀表自己製造噪音);提示只在無更急事項時顯示;`FLIGHTWAKE_NO_UPDATE_CHECK=1` 可關。
4. **update = 偵測 + force 的語法糖**,但價值在「不用記自己當初裝了什麼」:lang 讀 marker、statusline 讀 settings、private 讀 exclude。[[dogfood-dual-copy-drift]] 的跨 repo 手動 cp 苦工由此退役(開發期本 repo 的 hooks/ 源頭 vs .flightwake/ 安裝副本雙份仍在,該條 TRAPS 維持 active)。
5. 發 Release 時 gh 回「workflow scope required」(v0.8.2 同帳號可發)——切 kaiwutech-TW 帳號發成後切回;根因未查,暫記現象。
